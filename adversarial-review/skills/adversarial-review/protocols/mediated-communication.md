# Mediated Communication Protocol

## Purpose

Ensure agents never see raw output from other agents. All inter-agent communication passes through the orchestrator, which validates and sanitizes content before sharing. This prevents one compromised agent from influencing another through its output.

## Core Rule

**Agents never see raw output from other agents.** The orchestrator extracts validated findings, strips intermediate reasoning, and wraps field values in isolation markers before sharing.

## Provenance Markers

Every finding shared between agents carries a provenance marker:

```
[PROVENANCE::Specialist_Name::VERIFIED]
```

The orchestrator adds these markers after validation. They indicate:
- Which specialist produced the finding
- That the finding passed validation (`scripts/validate-output.sh`)

Provenance markers are metadata for the orchestrator. They must **never** appear inside the content of finding fields (Title, Evidence, Recommended fix). The validation script checks for this and rejects any finding that contains provenance or field marker patterns in its free-text fields.

## Field-Level Isolation

When sharing a finding's fields with another agent, each field value is wrapped in isolation markers generated by `scripts/generate-delimiters.sh`:

```
[FIELD_DATA_<hex>_START]field value here[FIELD_DATA_<hex>_END]
```

This prevents a crafted field value from breaking out of its boundary and being interpreted as a different field or as instructions.

## Validation Gate

Before any finding is shared between agents, it passes through `scripts/validate-output.sh`, which checks:

1. **Schema compliance** — all required fields present, correct format
2. **Injection detection** — free-text fields scanned for instruction-like patterns
3. **Marker contamination** — rejects findings containing `[PROVENANCE::` or `[FIELD_DATA_` patterns in field content

Only findings that pass all checks are shared. Failed findings are logged and excluded.

## What Is Never Shared

The following are never passed between agents:

- **Intermediate reasoning** — an agent's chain-of-thought or internal analysis
- **Self-refinement drafts** — earlier iterations of findings before convergence
- **Raw output** — the complete, unprocessed response from any agent
- **Validation errors** — details about why other agents' findings were rejected

Only validated, sanitized, final-iteration findings are shared across agent boundaries.

## Communication Flow

```
Agent A produces output
       |
       v
Orchestrator validates via validate-output.sh
       |
       v
Failed findings excluded, logged
       |
       v
Passing findings wrapped with provenance + field isolation markers
       |
       v
Sanitized findings shared with Agent B
```

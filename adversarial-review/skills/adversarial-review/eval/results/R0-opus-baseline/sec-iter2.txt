Source Trust: Privileged
File: internal/controller/services/auth/auth_controller_actions.go
Lines: 63-65
Title: Operator precedence bug in system:authenticated guard allows bypassing admin group protection
Evidence: In bindRole(), the condition `if roleName == "admingroup-role" && e == "system:authenticated" || e == ""` has an operator precedence issue. Due to && binding tighter than ||, this evaluates as `(roleName == "admingroup-role" && e == "system:authenticated") || (e == "")`. The e == "" check applies unconditionally to ALL roles, not just admin roles. The same pattern exists in bindClusterRole(). While the CRD-level validation provides a first line of defense, this controller-level guard is the last line of defense and could be bypassed if the webhook is unavailable.
Recommended fix: Add parentheses to clarify intent: `if roleName == "admingroup-role" && (e == "system:authenticated" || e == "")`.

Finding ID: SEC-002
Specialist: Security Auditor
Severity: Important
Confidence: High
Source Trust: Internal
File: pkg/cluster/cert.go
Lines: 106-118
Title: Self-signed certificate generated as CA with overly permissive key usage enables certificate forgery if key is compromised
Evidence: In generateCertificate(), the certificate template sets IsCA: true and includes KeyUsageCertSign. This certificate is used as a TLS server certificate (it self-signs itself). A TLS server certificate should never be a CA certificate. If the private key is compromised, an attacker could use it to sign arbitrary certificates.
Recommended fix: Set IsCA: false, remove KeyUsageCertSign from KeyUsage.

Finding ID: SEC-008
Specialist: Security Auditor
Severity: Minor
Confidence: High
Source Trust: Internal
File: pkg/cluster/cert.go
Lines: 91-93
Title: Certificate serial number has artificially limited entropy range
Evidence: Serial number generated as rand.Int(rand.Reader, big.NewInt(time.Now().UnixNano())). While crypto/rand is used, the upper bound is ~61 bits (time.Now().UnixNano()). RFC 5280 recommends at least 20 bytes (160 bits) of entropy.
Recommended fix: Use full 128-bit random value: rand.Int(rand.Reader, new(big.Int).Lsh(big.NewInt(1), 128)).

Finding ID: SEC-010

Severity: Minor

Source Trust: Internal
File: internal/controller/dscinitialization/utils.go
Lines: 262-275
Title: GenerateRandomHex produces half the expected byte count due to integer division
Evidence: GenerateRandomHex() computes numBytes := length / 2. Called with length=32, it generates 16 bytes (128 bits). The function name and parameter semantics are misleading, which could lead future callers to get less entropy than expected.
